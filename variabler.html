En int (32 bitar) kan innehålla värden från −2,147,483,648 till 2,147,483,647. 16-bitars kan ha värden från −32,768 till 32,767. 1 byte (8 bitar) kan ha värden från 0 till 2552.

float och double finns till för att lagra decimaltal i variabeln. Storleken skiljer åt hur mycket som kan lagras. Genom att skriva f i slutet av talet indikeras det.

En textsträng ses enklast som en sekvens med tecken, oftast alfabetiska sådana. En variabel av typen string kan i C# innehålla text. Allt som skrivs in i Console.WriteLine() är eller omvandlas till en textsträng.

Char är inte lika omfattande som string då den bara kan innehålla ett tecken. Varje bokstav som skrivs i char representeras av en siffra, något som kallas i UNICODE-tabellen. Det är en tabell som innehåller de flesta tecken som finns i världen och alla dess alfabet. Varje tecken har en siffra som motsvarar tecknet. Siffran 97 motsvarar "a", 98 "b" osv. Därför står det också unicode i tabellen.

En variabel av typen bool kan endast innehålla 1 av 2 värden, true eller false. Ger mest användning i samband med if-satser och loopar.

Variabler är föränderliga men det finns också konstanter i C#. De är oföränderliga variabler. De görs om till en konstant genom att const sätts framför. Variabelns värde måste anges för att kompilatorn inte ska klaga. Konstanter anges främst för att man ska kunna hålla koll på sina variabler.

Console.WriteLine() är till för att skriva ut saker på skärmen och för att läsa in data från användaren körs metoden Console.ReadLine(). Texten som användaren skriver in sparas i en string-variabel som vi kallar readText och sedan skrivs ut. Här använder vi oss av metoden Console.Write() istället för Console.WriteLine() då skillnaden är att den förstnämnda inte lägger in någon blankrad efter texten.

Det går att konvertera variabler, t.ex. om vi vill läsa in ett heltal från en användare. För att läsa in något behöver vi använda oss av Console.ReadLine, men den ger oss bara strängar. Då måste vi ersätta string till int. Det gör vi med metoden Convert.ToInt32 ();. Sedan anges variabeln strNr som är en blandning mellan string och nr.

Anledningen till att vi skriver in Convert.ToInt32 är att CLI int. i C# heter Int32 i CLI

Ett annat sätt att ändra variabler är genom typomvandling. Det är användbart om man vill tilldela en variabel av ett värde som fås av en annan slags variabeltyp. Komplilatorn ändrar automatiskt från t.ex. int nr till string. Det går inte att göra tvärtom utan då får man konvertera. Små saker får plats i stora men inte tvärtom. Det går att tvinga kompilatorn att godkänna konvertering från t.ex int till byte, läs mer på s. 54. I binär kod krävs det 20 bitar att skriva talet 999999: 1111 0100 0010 0011 1111. 63 skrivs med 6 bitar: 11 1111 eller 8 bitar: 0011 1111. En byte bryr sig bara om de 8 första bitarna av talet 999999, dvs. 1111 0011 och ger då talet 63. En typkonvertering användes även i datatypen char

En if-sats är en typ av villkorssats. Det är den enda villkorssatsen som behöver läras för att ett språk ska vara turingkomplett. Ofta nämns villkorssatser med namnet selektion. Ett program behöver göra olika val beroende på olika paramterar och värden på saker och ting och då kan if-satser användas. Med hjälp av if-satser kan olika värden jämföras med varandra.

Villkorssatsen == används för att jämföra om något är exakt lika med något annat. Det är skillnad på jämförelseopartorn == och tilldelningsoperatorn = då den sistnämnda tilldelar variabeln ett värde.

En if-sats kan skrivas i pseudokod vilket sedan översätts till programmeringsspråk. Exempel på pseudokod är:

IF temperature is 100

write "It's boiling!" on the screen.

Det vi vill kontrollera, om temperaturen är 100 grader, ligger efter if (). Därefter anges ingen ; utan istället {}. Däri ligger den kod vi vill ska utföras om villkorssatsen stämmer.

Att använda en if-sats utan något mer gör att vi kör en stycke kod (den mellan hakparanteserna) om villkorssatsen visar sig stämma. Annars gör vi ingenting och ifylls ett resultat som diffar mot det förväntade slutar programmet att köras. Om vi vill att något annat ska köras använder vi oss av villkorssatsen else

Vill man ha ytterligare ett alternativ kan else if läggas in. Den skrivs som pseudokod mellan if och else. Det går att använda sig av else if utan else.

Och-operatorn && kan användas om man vill kontrollera att 2 eller flera saker stämmer samtidigt.

Eller-operatorn || fungerar på samma sätt som och-operatorn. Istället för att båda värdena måste stämma räcker det med att ena värdet gör det.

Om man vill ha en if-sats inuti en annan if-sats är det fullt möjligt och det kallas nästlade if-satser

Switch är en typ av selektion som används istället för att fylla upp med en massa else if vid olika tänkta utfall. Det går till i följande steg:

  1. En heltalsvariabel skapas som vi tilldelar ett värde
  2. Ordet switch används följt av variabelnamnet
  3. Beroende på innehållet i variabeln får vi 3 olika utfall:
    • De första 2 utfallen tar hand om värdet i variabeln om man skriver det värde vi tilldelat variabeln
    • Utfallet default tar hand om alla andra värden
Det som står efter case är inte ett kodblock med klammerparenteser utan med ordet break; som avbryter hela switch-satsen.

Loop kallas det när ett kodstycke upprepas hela tiden. Det kan även kallas slinga eller iteration. Kodstycket upprepas oftast så länge ett visst villkor är sant. While-loopen är den viktigaste och mest användbara loopen och den enda som är värd att kunna. Här beskrivs den i pseudokod:

WHILE something SO
Do this
END WHILE

Ett annat exempel för vad vi konkret kan göra:

WHILE temperature is less than 100 SO
Add temperature by 1 and write line
Write "water is boiling"
END WHILE

Evighetsloop är när något loopar oändligt länge. Det skapas genom följande kodstycke:

while(true)
{
Console.WriteLine("loop...");
}
En loop fortsätter så länge det som står i parenterserna är sant, i det här fallet föralltid eftersom true alltid är sant. Med frasen break; avbryts den aktuella while-loopen, alltså den loop som körs inuti en annan loop. En while-loop är användbar om man vill att något ska köras tills användaren har gjort vissa val. Då bygger man bara på med if och else if tills användaren fyllt i alla sina val. Om något av valen leder en till ett ett annat program (man väljer t.ex. att spela ett spel) kommer man åter hamna i menyn när spelet är klart. Valen är alltså bara delar av while-loopen.

Närbesläktat med break är continue men skillnaden är att den inte avslutar loopen utan bara just det varvet av loopen och går vidare till nästa. varv

Med do while slipper man skriva ut samma kodstycke 2 gånger eftersom vi kan välja vilken kod den ska loopa tills den nått önskat resultat och det blir snyggare kod.

    Pseudokod

    Pseudo betyder falsk eller istället för och pseudokod är all kod som är skriven på mänskligt språk och inte knutet till ett speciellt programmeringsspråk. Pseudokod är vad vi anger när vi skriver ut vad som ska ske i ett program och används som instruktioner. Viktigt att använda rätt ord, t.ex.:
    IF something is yellow
    DO ask for another input
    ELSE finish menue
     Nu vet vi att vi ska infoga if-satsen och om användaren anger "yellow" ska vi be om mer inmatad data, annars ska menyvalet avslutas.
    Det finns ingen standard för pseudokod så det kan vara svårt att tolka. Viktigt är att alltid skriva på engelska.
    Det är fullt tillåtet att använda sig av andras smarta lösningar, algoritmer, som är skrivna i pseudokod.
